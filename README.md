## 🚀 프로젝트 실행 방법

### 사전 요구사항

- Node.js 20.x 이상
- yarn 또는 npm

### 환경 변수 설정

프로젝트 루트에 `.env` 파일을 생성하고 다음 환경 변수를 설정합니다:

```env
VITE_OPENWEATHER_API_KEY=your_openweather_api_key
VITE_KAKAO_REST_API_KEY=your_kakao_rest_api_key
VITE_KAKAO_JAVASCRIPT_KEY=your_kakao_javascript_key
```

### 설치 및 실행

```bash
# 의존성 설치
yarn install

# 개발 서버 실행
yarn dev

# 빌드
yarn build

# 빌드된 파일 미리보기
yarn preview

# Storybook 실행
yarn storybook

```

## ✨ 구현한 기능

### 1. 위치 기반 날씨 정보

- **현재 위치 날씨**: Geolocation API를 활용한 현재 위치의 실시간 날씨 정보 제공
- **시간별 예보**: 3시간 단위 날씨 예보를 가로 스크롤 카드로 시각화
- **날씨 아이콘**: Lucide React 아이콘으로 날씨 상태를 직관적으로 표시 (맑음, 흐림, 비, 눈 등)
- **치과 방문 권장 메시지**: 현재 날씨 조건에 따른 치과 방문 추천 메시지 제공

### 2. 장소 검색

- **통합 검색**: 행정구역(제공된 JSON 데이터) + 카카오맵 API 결과 통합 검색
- **초성 검색**: `korean-search-utils` 라이브러리를 활용한 한글 초성 검색 지원 (예: "ㅅㅇ" → "서울")
- **우선순위 정렬**: 즐겨찾기 → 행정구역 → 카카오맵 순으로 검색 결과 정렬
- **무한 스크롤**: 검색 결과 렌더링 최적화를 위한 무한 스크롤 구현

### 3. 주변 치과 정보

- **1km 반경 검색**: 선택한 위치 기준 1km 이내 치과 목록 표시
- **거리순 정렬**: 가까운 순서대로 치과 목록 정렬
- **치과 상세 정보**: 주소, 전화번호, 거리 정보 및 전화하기/길찾기 기능
- **카카오맵 연동**: 치과 위치를 지도에 마커로 표시

### 4. 즐겨찾기

- **장소 저장**: 최대 6개까지 자주 찾는 장소 저장
- **닉네임 설정**: 저장된 장소에 별칭 부여 가능
- **주소 기반 중복 체크**: 동일 주소 중복 저장 방지
- **빠른 검색**: 검색어 입력 시 즐겨찾기 우선 표시

### 5. 반응형 UI/UX

- **데스크톱**: 좌측 패널 + 우측 지도 레이아웃
- **모바일**: 전체화면 지도 + 드래그 가능한 바텀시트
- **키보드 네비게이션**: 검색 결과 화살표 키 탐색 지원

#### 반응형 기준

- **기준**: 가로 너비 768px
  - **모바일**: 768px 미만
  - **데스크탑**: 768px 이상

위 기준에 따라 레이아웃(패널 분리, 지도/사이드바 배치)과 상호작용(바텀시트, 드롭다운 동작 등)을 분기 처리합니다.

## 🧠 기술적 의사결정 및 이유

### 1. shadcn/ui 기반 디자인 시스템

**결정**: Tailwind CSS 기반의 shadcn/ui 컴포넌트를 베이스로 커스텀하여 사용

**이유**:

- shadcn/ui는 컴포넌트 코드를 직접 소유하여 자유로운 커스터마이징 가능
- Radix UI 기반으로 접근성(a11y)이 보장됨
- Tailwind CSS와 완벽한 호환으로 일관된 스타일링 가능
- 번들 사이즈 최적화 (사용하는 컴포넌트만 포함)

### 2. korean-search-utils를 활용한 검색 기능

**결정**: 직접 개발한 `korean-search-utils` 라이브러리를 사용하여 초성 검색 구현

**이유**:

- 한글 초성/중성/종성 분리 및 조합 알고리즘 적용
- "ㅅㅇ"으로 "서울", "ㄱㄴ"으로 "강남" 검색 가능
- 한글 특성에 최적화된 검색 경험 제공

### 3. 검색 결과 우선순위 시스템

**결정**: 즐겨찾기 → 행정구역 → 카카오 API 순으로 정렬

**이유**:

- 사용자가 자주 찾는 장소(즐겨찾기)를 최우선으로 노출
- 제공된 행정구역 JSON 데이터를 API 결과보다 우선 표시하여 정확도 향상
- 카카오맵 API 결과는 보완적 역할로 다양한 검색 결과 제공

### 4. 카카오맵 API + Geolocation 통합

**결정**: 카카오맵 JavaScript SDK와 Web Geolocation API를 통합하여 위치 기반 서비스 구현

**이유**:

- 카카오맵: 국내 지도 서비스 중 가장 정확한 한국 지역 데이터
- Geolocation API: 별도 라이브러리 없이 브라우저 기본 기능으로 현재 위치 획득
- 마커 클러스터링, 줌 컨트롤 등 지도 인터랙션 구현

### 5. 날씨 기반 치과 방문 추천 메시지

**결정**: OpenWeatherMap API 데이터를 분석하여 치과 방문 적합성 메시지 제공

**이유**:

- 단순 날씨 정보 제공을 넘어 서비스의 핵심 가치(치과 방문) 연결
- 온도, 날씨 상태, 가시거리 복합적 조건 분석
- 사용자에게 실질적인 의사결정 도움 제공

### 6. FSD (Feature-Sliced Design) 아키텍처

**결정**: FSD 아키텍처 패턴을 적용하여 프로젝트 구조화

**이유**:

- 기능 단위로 코드를 분리하여 유지보수성 향상
- 명확한 의존성 방향 (shared → features → widgets → pages)
- 새로운 기능 추가 시 영향 범위 최소화

## 📁 프로젝트 구조

```
src/
├── app/                    # 앱 진입점
├── pages/                  # 페이지 컴포넌트
│   ├── SearchPage.tsx      # 메인 검색 페이지
│   ├── LocationDetailPage.tsx  # 위치 상세 페이지
│   └── FavoritesPage.tsx   # 즐겨찾기 페이지
├── widgets/                # 독립적인 UI 블록
│   ├── weather/           # 날씨 관련 위젯
│   ├── clinic/            # 치과 관련 위젯
│   ├── search/            # 검색 관련 위젯
│   ├── favorites/         # 즐겨찾기 위젯
│   └── map/               # 지도 위젯
├── features/               # 비즈니스 로직
│   ├── weather/           # 날씨 기능 (hooks)
│   ├── clinic/            # 치과 기능 (hooks)
│   └── search/            # 검색 기능 (hooks, components)
├── shared/                 # 공통 모듈
│   ├── api/               # API 클라이언트
│   ├── ui/                # 공통 UI 컴포넌트
│   ├── lib/               # 유틸리티 함수
│   ├── hooks/             # 공통 훅
│   ├── store/             # 전역 상태 (Zustand)
│   ├── types/             # 타입 정의
│   └── data/              # 정적 데이터 (행정구역 JSON)
└── assets/                 # 정적 리소스
```
